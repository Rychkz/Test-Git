nel que se truene

Rules

-> No olvidar hacer un staged antes del commit

-> Seguir el siguiente flujo de trabajo 
    -> 1 guardar cambios 
    -> 2 realizar el staged -> recomendado usar la opción staged changes en lugar de Staged all changes
    -> 3 realizar el commit con comentarios
    -> 4 NO OLVIDAR HACER EL ###### PUSH #####
    -> Una vez solicitado y aprobado el pull request deberas elminar la rama de manera local, si aun no recibes la aprobación conserva la rama hasta obtener comentarios.
